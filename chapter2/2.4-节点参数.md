# 2.4 节点的参数

接下来我们将要学习节点的参数



## 2.4.1 参数是什么

**参数（Parameter）**是**节点**对外暴露的**动态配置项**，你可以在运行时修改参数值，从而改变节点的行为，无需重新编译或重启节点。

想象这样一个场景：你在调试一个PID控制器，每次调整参数都需要修改源码、重新编译、重新运行——效率极低。如果将PID的三个增益值声明为参数，你就可以直接在命令行中实时修改，节点会立即响应新的配置。

参数的应用场景非常广泛：**控制循环频率**、**设置阈值**、**启用/禁用功能开关**等。

ROS2参数支持多种数据类型：

1. **整型（integer）**
1. **浮点型（double）**
1. **布尔型（bool）**
1. **字符串（string）**
1. **数组（byte_array, bool_array, integer_array, double_array, string_array）**

参数既可以保存在节点内部，也可以持久化到YAML文件中，便于批量配置和版本管理。

通俗的讲，参数就像手机里的“设置”——你可以随时改铃声大小，不用重启手机。

## 2.4.2 声明和获取参数

那我们该如何使用参数呢，先了解一下参数的类型：

参数可以是**整数(int)、浮点数(float)、布尔值（bool）、字符串(string)**，甚至**数组**，这些基本够用了，那我们跟着流程走，就可以获取一个可灵活修改的参数了。

1.创建**变量（param_name）**给一个初始值，一般都是在私有权限下创建

2.声明参数，在构造函数中进行

```c++
this->declare_parameter<参数类型>(" param _name", param_name);
```

3.调用参数

```c++
this->get_parameter("param _name ", param _name);
```

4.然后运行节点，在终端中修改节点

```
ros2 run turtlesim turtlesim_node --ros-args -p < param_name >:=<value>
```

5.实例

```
ros2 run turtlesim turtlesim_node --ros-args \
-p background_r:=255	# 设置背景色红色通道为100\
-p use_sim_time:=true 	# 启用仿真时间 
```

## 2.4.3 操作参数的命令

1、查看节点的所有参数

```
ros2 param list
```

2、获取某个参数的值

```
ros2 param get <node_name> <param_name>
```

3、设置参数（立即生效）

```
ros2 param set <node_name> < param_name > <value>
```

4、把参数保存到文件

```
ros2 param dump <node_name>
```

5、启动时加载参数文件

```
ros2 run <packages_name> <node_name> --ros-args --params-file param.yaml
```

参数机制让节点变得灵活可配置，但普通节点的生命周期过于简单——启动即运行，关闭即销毁。
