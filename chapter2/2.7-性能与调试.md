# 2.7 节点性能与调试

节点跑得快不快的问题解决了。但还有一个更实际的问题：

节点出问题了怎么办？

ROS2提供了一套完整的调试工具箱，帮助你快速定位和解决节点运行中的问题。



## 2.7.1 常用调试命令

1、列出所有运行中的**节点**

```
ros2 node list
```

2、查看节点**详细信息**（订阅/发布的**话题**、**服务**等）

```
ros2 node info <node_name>
```

3、 列出所有运行中的**话题**

```
ros2 topic list
```

4、 查看**话题详细内容信息**

```
ros2 topic echo <topic_name>
```

5、查看**话题的发布频率**

```
ros2 topic hz <topic_name>
```

6、查看**话题的带宽占用**

```
ros2 topic bw <topic_name>
```

7、图形化显示**节点和话题的关系图**

```
rqt_graph
```

**rqt_graph** 是你的好朋友。当你不清楚节点之间的连接关系时，打开它，整个系统的通信拓扑一目了然，比阅读代码高效得多。

此外，**rqt** 是基于Qt的GUI框架，本身是一个插件化平台。通过安装不同的插件，它可以变身为各种调试神器（如日志查看器、绘图工具、参数编辑器），**功能强大且使用便捷**，值得课后深入研究。
