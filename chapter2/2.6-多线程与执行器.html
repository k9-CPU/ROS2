2.6 节点的多线程与执行器

2.6 节点的多线程与执行器

如果你的节点需要同时处理多个耗时任务，单线程执行可能会导致回调阻塞。这时就需要引入多线程机制。

但在讨论多线程之前，我们必须先理解一个核心概念：执行器（Executor）。

2.6.1 什么是执行器（Executor）

一个节点内部通常注册了多个回调函数——订阅收到消息时触发一个，定时器到期时触发一个，服务请求到来时再触发一个。问题是：谁来调用这些回调？按什么顺序？一次能执行几个？

这就是执行器（Executor）的职责，它是ROS2中负责调度节点回调函数（如订阅回调、定时器回调、服务回调）的机制。它从回调队列中取出任务，并分配给线程执行。

而在本章的入门篇动手写第一个节点中

rclcpp::spin(node)

这行代码实际上创建了一个单线程执行器（SingleThreadedExecutor），所有回调被塞入同一个线程中顺序执行。

简单、安全、无并发冲突，对于小型节点来说完全够用。那单线程执行器有什么缺点呢？

假设你的节点有两个回调：一个处理激光雷达（耗时0.1秒），一个处理相机图像（耗时0.5秒）。

单线程执行器会这样干：

单线程执行器会串行处理，相机回调执行期间，激光雷达回调会被阻塞，导致数据丢失或延迟。

当单线程成为性能瓶颈时，就需要多线程执行器登场了

2.6.2 多线程执行器

多线程执行器（MultiThreadedExecutor） 允许将不同的回调分配到不同的线程中并发执行，从而避免单个耗时回调阻塞整个节点。

我们如何使用多线程处理器呢？

#include "rclcpp/rclcpp.hpp"
#include "rclcpp/executors/multi_threaded_executor.hpp"

class MyNode : public rclcpp::Node
{
public:
    MyNode() : Node("my_node")
    {
    RCLCPP_INFO(this->get_logger(), "Node start successfully!!!");
    // 创建多个订阅者或定时器，它们会在不同线程中并发执行
    }
};

int main(int argc, char** argv)
{
    rclcpp::init(argc, argv);

    auto node = std::make_shared<MyNode>();

    // 创建4线程的执行器
    rclcpp::executors::MultiThreadedExecutor executor(
        rclcpp::ExecutorOptions(),  // 执行器选项
        4);                          // 线程数量

    executor.add_node(node);
    executor.spin();  // 阻塞直到关闭

    rclcpp::shutdown();
    return 0;
}

多线程执行器的工作机制是：每个回调被放入线程池中，由空闲线程抢占执行。激光雷达回调和相机回调可以真正并行运行，互不干扰。
